## 前言

目前大多数行业都与人工智能挂钩，需要了解以及学习人工智能的相关知识。

## 什么是人工智能？

[人工智能](https://baike.baidu.com/item/%E4%BA%BA%E5%B7%A5%E6%99%BA%E8%83%BD/9180)（Artificial Intelligence），英文缩写为AI。是研究、开发用于模拟、延伸和扩展人的智能的理论、方法、技术及应用系统的一门新技术科学。人工智能是[计算机科学](https://baike.baidu.com/item/计算机科学/9132?fromModule=lemma_inlink)的一个分支，它企图了解[智能](https://baike.baidu.com/item/智能/66637?fromModule=lemma_inlink)的实质，并生产出一种新的能以人类智能相似的方式做出反应的[智能机器](https://baike.baidu.com/item/智能机器/392924?fromModule=lemma_inlink)，该领域的研究包括[机器人](https://baike.baidu.com/item/机器人/888?fromModule=lemma_inlink)、[语言识别](https://baike.baidu.com/item/语言识别/1212421?fromModule=lemma_inlink)、[图像识别](https://baike.baidu.com/item/图像识别/6263637?fromModule=lemma_inlink)、[自然语言处理](https://baike.baidu.com/item/自然语言处理/365730?fromModule=lemma_inlink)和[专家系统](https://baike.baidu.com/item/专家系统/267819?fromModule=lemma_inlink)等。

> 简单来说，它能将”看起来智能的行为“系统地做出来并可复制、可规模化

## 人工智能概念详解

### 什么是LLM？

[大语言模型](https://www.runoob.com/ai-agent/ai-agent-llm.html)（Large Language Model，简称 LLM）是 AI Agent 的大脑，理解它是构建智能 Agent 的基础。大语言模型之所以能与你对话、写文章、编程，本质上是它在根据你给出的文本（提示），一个字一个字地**猜**出最合理的下文。简单来说，大语言模型是一个经过海量文本数据训练的深度学习模型，它能够理解和生成人类语言。大语言模型通过分析互联网上的海量文本，学习语言的统计规律，当收到输入时，根据学到的规律生成最合理的续写。

![](https://www.runoob.com/wp-content/uploads/2026/02/runoob-llm-work.png)

> 我的理解是它是一个会预测会输出的搜索引擎。

### 什么是Prompt？

[提示词](https://www.runoob.com/ai-agent/ai-agent-prompt.html)（Prompt）是指用户输入给 AI 模型的文本指令，用于引导模型生成特定类型的输出。一个好的 Prompt 设计可以显著提升 AI 输出的质量和准确性。

编写 Prompt 的核心原则包括：
- **清晰明确**：指令越具体，输出越符合预期
- **提供上下文**：给模型足够的背景信息
- **设定角色**：让模型扮演特定角色来回答问题
- **分步引导**：复杂任务拆解为多个步骤

> 简单来说，Prompt 就是你与 AI 对话时的"问法"，好的问题才能得到好的答案。

### 什么是 Token？

Token（令牌/词元）是 LLM 处理文本时的基本单位。一个 Token 可以是一个词、一个汉字的一部分，或者是一个标点符号。例如：
- 英文中，"Hello" 可能是一个 Token
- 中文中，"人工智能" 可能被切分成 2-4 个 Token

LLM 的上下文窗口以 Token 数量衡量，例如 4K、8K、128K 等，表示模型一次能处理的最大 Token 数。

> 理解 Token 有助于控制输入长度和成本，因为大多数 AI 服务按 Token 计费。

### 什么是 RAG？

[RAG](https://www.runoob.com/ai-agent/ai-agent-rag.html)（Retrieval-Augmented Generation，检索增强生成）是一种融合信息检索与文本生成的先进技术架构。它在生成回答前，先从外部知识库中检索出最相关的信息，再将这些内容作为上下文提供给 LLM，从而显著提升回答的准确性、相关性和时效性。

RAG 的核心流程：
1. **检索**：将用户问题转换为向量，在知识库中搜索最相关的内容
2. **增强**：将检索到的内容与原始问题拼接作为 Prompt
3. **生成**：LLM 基于增强后的上下文生成回答

> RAG 有效解决了 LLM 知识更新不及时和"幻觉"问题，是企业落地 AI 应用的关键技术。

### 什么是 AI Agent？

[AI Agent](https://www.runoob.com/ai-agent/ai-agent-intro.html)（人工智能代理）是一个能够自主感知环境、制定计划并执行行动的智能实体。与单纯的 LLM 不同，AI Agent 具备以下能力：
- **感知**：接收和理解用户输入及环境信息
- **规划**：拆解复杂任务，制定执行步骤
- **记忆**：存储历史对话和关键信息
- **工具使用**：调用外部 API、数据库等工具完成操作
- **自主执行**：按计划逐步完成任务

> AI Agent 是 LLM 从"聊天机器人"走向"智能助手"的关键进化形态。

### 什么是 Fine-tuning？

微调（Fine-tuning）是指在预训练的大语言模型基础上，使用特定领域的数据进一步训练，让模型更擅长某个特定任务或领域的知识。相比于从头训练一个模型，微调可以大幅节省时间和计算资源。

常见微调方式：
- **全量微调**：更新模型所有参数
- **LoRA**：仅更新少量参数的高效微调方法
- **QLoRA**：量化版本的 LoRA，进一步降低显存需求

> 微调适合需要模型深度掌握特定领域知识的场景，一般场景使用 Prompt 工程或 RAG 就足够了。

## 总结

本文介绍了人工智能领域的几个核心概念，从基础的 AI 定义到 LLM、Prompt、Token、RAG、AI Agent 和 Fine-tuning，这些概念构成了现代 AI 应用开发的知识基础。理解它们之间的关系：

```
AI（人工智能）
  └── LLM（大语言模型）
        ├── Prompt（提示词工程）
        ├── RAG（检索增强生成）
        ├── AI Agent（智能代理）
        └── Fine-tuning（模型微调）
```

随着 AI 技术的不断发展，这些概念也在持续演进。掌握这些基础知识，将帮助你更好地理解和使用 AI 技术。
